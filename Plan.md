# План тестирования мобильного приложения «Мобильный хоспис»

## 1. Границы тестирования (Scope) и план автоматизации
В рамках проекта проводится полное ручное тестирование. Для второй части диплома (автоматизация) выбраны критические сценарии (Smoke-тесты).


| Функциональный блок | Что тестируем (сценарий) | ID Тест-кейса | Автоматизация |
| :--- | :--- |:--------------| :--- |
| **Авторизация** | Вход с валидными данными | TC-01         | **Да** |
| **Авторизация** | Ошибка при пустых полях | TC-02         | Нет (Manual) |
| **Новости** | Просмотр списка новостей | TC-03         | **Да** |
| **Новости** | Разворачивание карточки новости | TC-04         | **Да** |
| **Новости** | Фильтрация по категории | TC-05         | Нет (Manual) |
| **Control Panel** | Создание новой новости | TC-06         | **Да** |
| **Control Panel** | Удаление новости | TC-07         | **Да** |
| **Цитаты** | Переход в раздел «Our Mission» | TC-10         | Нет (Manual) |
| **О приложении** | Просмотр версии ПО | TC-12         | Нет (Manual) |

## 2. Окружение (Test Environment)
Тестирование выполняется на эмуляторах среды Android Studio:
* **Основной:** Pixel 6, Android 13.0 (API 33) — для функциональных и автотестов.
* **Дополнительный:** Nexus 5, Android 9.0 (API 28) — для проверки адаптивности интерфейса.

## 3. Типы тестирования
* **Функциональное:** проверка CRUD новостей и работы логики авторизации.
* **UI/UX:** проверка верстки, шрифтов и корректности иконок.
* **Negative testing:** ввод некорректных данных, обработка ошибок.
* **Автоматизированное:** реализация Smoke-сценариев через Espresso.
